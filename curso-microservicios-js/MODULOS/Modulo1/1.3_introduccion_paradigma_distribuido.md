## рҹ§© ВҝQuГ© es una arquitectura distribuida?

Una **arquitectura distribuida** es aquella en la que **las funciones de una aplicaciГіn estГЎn separadas en componentes independientes** que se ejecutan en **procesos o servidores distintos**, pero que **cooperan entre sГӯ mediante comunicaciГіn por red** (HTTP, gRPC, mensajerГӯa, etc.).

рҹ‘ү En lugar de tener **una Гәnica aplicaciГіn que hace todo**, tenemos **varias piezas mГЎs pequeГұas**, cada una responsable de una tarea especГӯfica.

---

## вҡҷпёҸ Ejemplo visual

```

 в”ңв”Җв”Җ /users
 в”ңв”Җв”Җ /orders
 в”ңв”Җв”Җ /inventory
```

**Arquitectura distribuida (cada mГіdulo independiente):**

* Puede tener su propia base de datos.

## рҹ”— ComunicaciГіn entre servicios

Como los servicios ya no viven en el mismo proceso, deben **comunicarse por red**.

### рҹ”ё Ejemplo con Node.js (comunicaciГіn REST)

**Servicio de usuarios (`users-service`):**

```js
// users-service/index.js
import express from 'express';
const app = express();

app.get('/users', (req, res) => {
  res.json([{ id: 1, name: 'Ana' }]);
});

app.listen(3001, () => console.log('Users service en puerto 3001'));
```

**Servicio de pedidos (`orders-service`):**

```js
// orders-service/index.js
import express from 'express';
import axios from 'axios';
const app = express();

app.get('/orders', async (req, res) => {
  const { data: users } = await axios.get('http://localhost:3001/users');
  res.json([{ orderId: 100, user: users[0].name, total: 59.90 }]);
});

app.listen(3002, () => console.log('Orders service en puerto 3002'));
```

вһЎпёҸ AquГӯ `orders-service` **consume la API de `users-service`** vГӯa HTTP en lugar de llamar directamente una funciГіn local.

## рҹ§ұ Principios bГЎsicos del paradigma distribuido

| Principio                           | DescripciГіn                                                          |
| ----------------------------------- | -------------------------------------------------------------------- |
| **SeparaciГіn de responsabilidades** | Cada servicio cubre una parte funcional concreta del negocio.        |
| **ComunicaciГіn mediante red**       | Los servicios interactГәan con APIs o colas de mensajes.              |
| **Despliegue independiente**        | Se puede actualizar o reiniciar un servicio sin afectar a los demГЎs. |
| **Escalabilidad selectiva**         | Solo se escalan los componentes que lo requieren.                    |
| **Resiliencia**                     | Un fallo en un servicio no deberГӯa detener todo el sistema.          |

---

## рҹ§  Beneficios clave

| Beneficio                 | ExplicaciГіn                                                        |
| ------------------------- | ------------------------------------------------------------------ |
| **Escalado granular**     | Cada componente puede ajustarse segГәn su carga de trabajo.         |
| **Desarrollo paralelo**   | Distintos equipos pueden trabajar en servicios independientes.     |
| **Mantenibilidad**        | CГіdigo mГЎs acotado, fГЎcil de probar y desplegar.                   |
| **EvoluciГіn tecnolГіgica** | Cada servicio puede usar su propio stack (Node.js, Python, Go...). |

## вҡ пёҸ Retos de las arquitecturas distribuidas

Aunque solucionan problemas del monolito, tambiГ©n introducen nuevos desafГӯos:

| DesafГӯo                      | Ejemplo                                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------- |
| **Latencia de red**          | Las llamadas HTTP son mГЎs lentas que las llamadas locales.                            |
| **GestiГіn de errores**       | Si un servicio no responde, hay que manejar timeouts y reintentos.                    |
| **Coherencia de datos**      | Cada servicio puede tener su propia base de datos вҶ’ surge la вҖңconsistencia eventualвҖқ. |
| **Monitoreo y trazabilidad** | Se necesitan herramientas centralizadas de observabilidad.                            |

## рҹ”Қ AnalogГӯa docente

> Imagina una empresa:
>
> * En un monolito, **todo el personal trabaja en una misma oficina**.
> * En una arquitectura distribuida, **cada departamento tiene su propia sede**, pero todos **se coordinan mediante comunicaciones**.

Esta divisiГіn permite crecer mГЎs rГЎpido, pero requiere **organizaciГіn, normas y canales eficientes**.
